
# Markdown Grundlagen und Praxis

## Beschreibung

In dieser Übung lernst Du die Grundlagen von Markdown kennen und wendest diese direkt praktisch an. 

Wir wünschen Dir viel Erfolg bei der Bearbeitung dieser Aufgabe.

---

## Aufgabe

Bearbeite die folgenden Teilaufgaben Schritt für Schritt und festige das Gelernte durch praktische Anwendung. Die Reihenfolge sollte dabei eingehalten werden, da die Aufgaben aufeinander aufbauen und an Schwierigkeitsgrad zunehmen.

## Hilfsmittel

- [Visual Studio Code (VS Code)](https://code.visualstudio.com)
- <https://www.markdownguide.org>

## Zielsetzung

Nach Abschluss dieser Aufgabe kannst Du:

- Markdown sicher anwenden
- Strukturierte Dokumentationen erstellen
- Code sauber darstellen
- Markdown im Kontext von IT und Automatisierung nutzen
- Markdown für KI-Prompts einsetzen


## Zeitvorgabe

Gesamtdauer: ca. 8 Stunden

| Abschnitt                   | Dauer      |
|-----------------------------|------------|
| Block 1 – Einstieg          | 1 Std.     |
| Block 2 – Grundlagen        | 1,5 Std.   |
| Block 3 – VS Code           | 1 Std.     |
| Block 4 – Tabellen          | 1,5 Std.   |
| Block 6 – AI Prompt         | 1 Std.     |
| Block 7 – Fehleranalyse     | 1 Std.     |
| Block 8 – Abschluss         | 1 Std.     |


## Teilaufgaben

### Block 1 – Einstieg (ca. 60 Minuten)

1. Recherchiere kurz:
   - Was ist Markdown?
   - Wofür wird es verwendet?
   - Welche Editoren können zum Schreiben von Markdown verwendet werden?
   - Mit welchen Programmen kann Markdown in gerenderter Form angezeigt werden?

2. Erstelle eine erste Markdown-Datei (`Steckbrief.md`):
   - Überschrift: Dein Name
   - Unterüberschrift: Ausbildung Fachinformatiker Systemintegration
   - Kurze Beschreibung von Dir (2–3 Sätze)
   - Liste Deiner bisherigen IT-Kenntnisse

📖 **Tipp:**
- Nutze die oben genannte Webseite <https://www.markdownguide.org>, um Dich mit der dafür benötigten Synthax vertrau zu machen.<br>
⚠️ **Vermeide es aber, Dich von den noch nicht benötigten Abschnitten ablenken zu lassen**

✅ **Bonus:**
- Füge einen Link zu einer Website ein (z.B. Dein Ausbildungsbetrieb)

---

### Block 2 – Grundsyntax (ca. 90 Minuten)

Erweitere Deine Datei um folgende Inhalte:

1. Textformatierung:
   - Fett
   - Kursiv
   - Kombination aus beidem

2. Listen:
   - Aufzählungsliste
   - Nummerierte Liste
   - Verschachtelte Liste

3. Code:
   - Inline-Code
   - Codeblock mit Kennzeichnung `powershell`

**Beispiel** (selbst erstellen!):
- Ein einfacher PowerShell-Befehl (`Get-Process`)

---

### Block 3 – Arbeiten mit VS Code (ca. 60 Minuten)

1. Öffne Deine Markdown-Datei in VS Code

2. Aktiviere die Vorschau (Markdown Preview)

3. Teste:
   - Änderungen im Text → direkte Auswirkung in der Vorschau

4. Speichere Deine Datei

✅ **Zusatz:**
- Öffne Vorschau und Editor nebeneinander

---

### Block 4 – Tabellen und Struktur (ca. 90 Minuten)

Erstelle eine neue Datei: `tools.md`

Inhalt:

1. Überschrift: „Meine wichtigsten IT-Tools“

2. Tabelle mit folgenden Spalten:
   - Tool
   - Zweck
   - Bewertung (1–5)

3. Füge mindestens 5 Tools hinzu:
   - z. B. VS Code, PowerShell, Teams, GitHub

---

### Block 5 – KI Prompt Struktur (ca. 60 Minuten)

Erstelle eine Datei: `ai-prompt.md`

Schreibe einen strukturierten Prompt in Markdown:

Beispielstruktur:

    # Aufgabe
    Erkläre einen PowerShell-Befehl

    # Eingabe
    (Get-Process)

    # Anforderungen
    - Einfache Sprache
    - Beispiel hinzufügen
    - Schrittweise Erklärung

---

### Block 6 – Fehleranalyse (ca. 45 Minuten)

Kopiere folgenden fehlerhaften Inhalt und behebe ihn:

```
#Titel
##Untertitel
-Item1
-Item 2
**Text fett
`Codeblock
```

Ziel:
- Korrekte Darstellung in der Vorschau

---

### Block 7 – Abschlusschallenge (ca. 60 Minuten)

Erstelle eine vollständige Datei: `readme.md`

Thema:
„Ein fiktives IT-Tool Deiner Wahl“

Inhalt:
- Beschreibung
- Features (Liste)
- Installation (Schritte)
- Beispiel (Codeblock)
- Tabelle (z. B. Versionsübersicht)
- Link zu Website

✅ Ziel:
Eine sauber strukturierte, gut lesbare Dokumentation
